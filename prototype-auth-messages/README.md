# Prototype 2: Auth Flow Messaging Improvements

Flow simulator covering all 11 auth screens. Compare current developer-jargon copy against proposed user-friendly messaging for field workers.

See the [root README](../README.md) for full documentation, setup instructions, and how to use.

## Quick Start

```bash
npm install
npm run dev
# Open http://localhost:5174
```

## What to try

1. **Toggle Current/Proposed** to compare jargon vs user-friendly copy on each screen
2. **Pick scenarios** (connection drops, session expired, slow 3G) to see edge case messaging
3. **Switch First-time/Returning** to see personalization differences
4. **AuthLoadingScreenV2** is the most important screen -- try all 5 scenarios
5. **AccessModal** shows the admin action card design for permission errors
